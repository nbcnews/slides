what is git?

(and GitHub?)

Git vs github

  • Git is a free, open source 
version control system

  • GitHub is a company that offers
hosting of Git repositories (plus extras)

Other differences

  • Forks
  • Branching, merging is less painful
  • One folder per repository
  • Doesn't lock files

repos, forks, 

branches

REPOS

  • Anyone in UX could merge anytime
  • Limited number of private repos
  • Unlimited public repos

FORKS

  • Copies entire repo, including branches
  • Happens on GitHub, not locally
  • Great for creating derivative works
  • Anyone can do it, no authorization needed
  • Pull-request to get changes back into master

BRANCHES

  • Copies repo locally
  • Must be authorized
  • Easy to merge back to master via pull-request
  • We'll be using these a lot

issues

Bug tracking / todos

  • Repo-specific
  • Automatically creating for pull requests
  • Similar to todo items in BaseCamp
  • Can assign to someone, add comments

discussions

  • Great for gathering opinions
  • Can 'tag' specific people in comments
  • Notifications make them easy to track
